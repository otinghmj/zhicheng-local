# 模式：deep-prep-portfolio — 小作品框架

## 定位

面试前的作品武装模块。AI 结合 JD 核心技能，为候选人设计一个**可以在 1-3 天内完成的小作品**，帮候选人在面试中拿出实际产出物，而不只是口头描述。

**重要边界**：AI 只给框架和规格，**不生成代码、不生成内容**。候选人负责实际完成。

目标：候选人面试前有一个真实的、专门为这家公司/岗位设计的小作品，可以当场演示或在简历 / cover letter 中引用。

---

## 读取材料

1. 对应的评估报告（`reports/{子目录}/{###}-{company}-{date}.md`）— A/B 块，提取 JD 核心技术栈和行业场景
2. `cv.md` — 候选人已有项目，避免重复，寻找延伸点

---

## 执行步骤

### 第一步：分析作品方向

结合 JD 核心技能和候选人背景，识别最适合做小作品的切入点：

**优先选择**：
- 候选人有 50-70% 能力储备（不是从零学，也不是展示已熟悉的东西）
- 成品可以快速演示（CLI 工具 / 简单 Web 页面 / Jupyter notebook / 数据可视化报告）
- 贴合这家公司的业务场景（最好能让面试官直接代入）

### 第二步：输出 2-3 个作品候选

每个候选按以下结构描述：

```
【作品名】{简短名称}
【核心价值主张】一句话说明：这个作品证明了候选人哪个能力
【与 JD 的关联】直接对应 JD 中的哪条职责/要求
【业务场景设定】假设的业务背景（贴近这家公司）
【功能规格】
  - 输入：{用户/数据输入什么}
  - 核心处理：{主要逻辑或算法，不超过 3 步}
  - 输出：{产出什么，怎么展示}
  - 技术栈：{具体用哪些库/工具，不要泛泛}
【交付物清单】
  - {文件/模块1}：{作用}
  - {文件/模块2}：{作用}
  - README.md：包含业务背景、思路说明、运行方式
【完成时间估算】{候选人按现有能力大概需要多少天}
【面试时怎么用】{演示哪个部分 / 在 cover letter 中怎么提 / 适合放哪个位置}
```

### 第三步：候选人选择 + 细化规格

候选人选定其中一个方向后，AI 进一步细化：

- 给出 README 的推荐结构（候选人自己写内容）
- 给出数据来源建议（公开数据集、mock 数据、手工整理）
- 给出评估标准（面试官会怎么看这个作品）

---

## 输出格式

```
═══ 小作品框架：{公司} — {岗位} ══════════════════════

以下是 {N} 个候选方向，请选择一个继续：

━━━ 方案一 ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【{作品名}】
  核心价值：……
  JD 关联：……
  技术栈：……
  完成时间：约 {N} 天
  交付物：……

━━━ 方案二 ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
……

输入 1/2/3 选择一个方向，我给你细化完整规格。
```

选定后输出完整规格文档。

---

## 写入产出文件

候选人选定方向并确认规格后，将**框架文档**追加写入：

`interview-prep/{公司slug}-{岗位slug}-deep.md`

```markdown
## 模块5：小作品框架
> 生成日期：{YYYY-MM-DD}

### 选定方案

**{作品名}**

{完整的规格描述，含功能规格、交付物清单、README 结构建议、数据来源}

### 面试引用指南

{如何在 cover letter / 面试中提到这个作品，建议的一句话描述}
```

> **AI 只提供框架，不生成任何代码或内容。** 候选人按规格自行完成。
