# API 契约差距清单

> 核对日期：2026-06-11
> 核对基线：`docs/api-contract.yaml` 原有 20 个路径端点
> 页面范围：`web/design/` 中 7 个业务导航页面。目录内另外 5 个 HTML 属于设计系统、规划或说明文档，不纳入业务接口覆盖核对。

分类口径：下文“覆盖”按冻结后的 1.2 契约判断。“未覆盖”和“语义不符”保留原差距描述，但每条均标注批准后的处置归宿。

## 结论

- **已覆盖**：现有契约已覆盖应用记录、Pipeline 基础读写、评估报告、基础配置、状态枚举、扫描历史、批处理、脚本执行、PDF 单文件访问与 SSE 事件流。
- **1.2 已入约**：补齐报告章节、Pipeline 行字段、应用 join 字段、报告摘要字段、结构化脚本进度、取消任务、批量移除、新增故事、三份落账历史、PDF 列表、面试准备产物与新增投递记录。
- **明确不提供**：前端过滤/导出/聚合类能力不新增服务端端点；模板管理降级；故事编辑删除、PDF 删除/批量 ZIP、报告结构化对比列入二期 P2。

## 公共页头

### 覆盖

- 用户姓名、头像所需个人信息、`profile.yml` 来源：`GET /api/config/profile`
- 公司/职位搜索：可分别通过 `GET /api/applications?search=` 与报告列表筛选能力部分支持

### 未覆盖

- CRD、HSP、API 在线状态。**处置：后续指令吸收**，走 `POST /api/scripts/doctor` 的 `--json` 输出。
- 通知数量与通知列表。**处置：前端解决**，复用 `GET /api/activity-log`。
- 跨应用、Pipeline、报告的统一全局搜索。**处置：前端解决**，由前端 `⌘K` 聚合。

### 语义不符

- 当前搜索分散在各资源端点，不是页面表现出的统一搜索。**处置：前端解决**，不新增统一搜索端点。

## Dashboard

### 页面全部数据元素

- 今日日期、问候用户名
- 今日采集条数、较昨日变化
- Pipeline Pending、Processed、总计、最近处理时间、处理进度
- 待评估报告、投递中职位、面试中职位
- 运行中任务：平台、任务名、通道、当前步骤、页数进度、百分比、预计剩余时间
- 环境就绪状态：各平台连接、Chrome、浏览器驱动、OpenCLI、网络
- 最近采集任务：平台、关键词、城市、采集时间、采集条数、去重后、初筛通过、状态
- 去重历史：总条数、7 天新增、7 天跳过、更新时间
- 快捷操作记录、最近活动
- 系统状态：数据目录、配置文件、API、去重服务、评分模型

### 覆盖

- 用户信息：`GET /api/config/profile`
- 应用总量和状态计数：`GET /api/applications/metrics`
- Pipeline 基础数量：`GET /api/pipeline` 可由列表计算
- 报告数量：`GET /api/reports`
- 批处理概况：`GET /api/batch`
- 脚本任务状态和实时事件：`GET /api/scripts/{jobId}/status`、`GET /api/events`
- 按日新增/跳过：`GET /api/scan-history?aggregate=daily`（本次新增）

### 未覆盖

- 环境与系统健康检查。**处置：后续指令吸收**，走 doctor 脚本 JSON 输出。
- 通知、快捷操作记录、统一最近活动流。**处置：已入约 1.2**，读取 `GET /api/activity-log`。
- 采集任务结构化历史与运行中任务列表。**处置：已入约 1.2**，历史读取 `GET /api/task-history`，运行中任务复用脚本状态。

### 语义不符

- `DashboardMetrics` 未直接定义“今日采集、待评估报告、投递中、面试中、较昨日变化”等页面指标。**处置：前端解决**，组合既有端点与 `GET /api/metrics-history`。
- 脚本状态只提供通用输出，无法稳定提供当前步骤、页数、百分比和预计剩余时间。**处置：已入约 1.2**，增加 `progress` 与 `logTail`；预计剩余时间由前端按速率计算。

## 采集任务

### 页面全部数据元素

- 运行中任务：任务名、平台、状态、进度数量/总数/百分比、运行时长、关键词、预计剩余时间
- 环境状态
- 今日去重：总扫描、新增候选、去重过滤、去重率
- 任务历史：任务名、平台、城市、状态、进度、新发现、去重率、开始时间、时长
- 新建任务：平台、关键词、城市代码、最大滚动轮数、高级选项
- 历史筛选：搜索词、平台、状态、日期范围

### 覆盖

- 启动已有采集脚本：`POST /api/scripts/{name}`
- 查询任务状态与事件：`GET /api/scripts/{jobId}/status`、`GET /api/events`
- 扫描明细和每日新增/跳过：`GET /api/scan-history`
- 采集配置：`GET/PUT /api/config/portals`

### 未覆盖

- 通用“新建采集任务”资源端点。**处置：设计稿降级**，按平台映射到既有 `POST /api/scripts/{name}`。
- 结构化任务历史、取消任务、重试任务。**处置：已入约 1.2**，增加 `GET /api/task-history` 与 `DELETE /api/scripts/{jobId}`；重试仍重新调用原脚本 POST。
- 环境健康检查。**处置：后续指令吸收**，走 doctor 脚本 JSON 输出。

### 语义不符

- `ScriptExecution` 不含平台、城市、关键词、进度计数、运行时长、新发现、去重率等字段。**处置：已入约 1.2**，实时进度进入 `ScriptExecution.progress`，完整任务参数与结果进入 `GET /api/task-history`。
- `scan-history.tsv` 记录职位去重历史，不等价于一次采集任务的运行历史。**处置：已入约 1.2**，新增独立 `data/task-history.tsv` 读取端点。

## 1.1 冻结决定

1. 契约版本冻结为 `1.1`，冻结日期为 `2026-06-11`。
2. 新增 `POST /api/ai-tasks`，白名单严格采用 `modes/` 下真实文件名；需求示例中的 `deep-prep-industry` 不存在，使用真实的 `deep-prep-immersion`。
3. AI 任务并发数固定为 1，执行方式参照 `batch/batch-runner.sh` 的 `claude -p` headless 模式。
4. AI 任务状态与完成通知不新增另一套协议，复用脚本状态端点与 SSE。
5. `GET /api/config/states` 已存在且语义等价，不重复新增。

## 1.2 修订记录

1. 契约版本修订并重新冻结为 `1.2`，冻结日期保持 `2026-06-11`。
2. 报告详情章节与真实报告对齐为 A-E、`overall`、`nextSteps`。
3. 补齐 Pipeline 真实行字段、platform 推断、解析降级形态及批量移除。
4. 增加应用只读 join 派生字段、报告摘要派生字段、结构化脚本进度、日志尾部与取消任务。
5. 新增故事追加、任务历史、活动历史、指标历史、PDF 列表、面试准备产物、新增投递记录端点。
6. 契约末尾加入“明确不提供”，冻结前端解决、设计稿降级与二期 P2 边界。
